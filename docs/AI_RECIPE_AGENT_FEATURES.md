# AIレシピエージェント機能まとめ

**Last Updated**: 2026-07-03  
**Status**: Active

## 概要

このアプリには、レシピ作成と既存レシピ改善を支援するマルチAIエージェント機能があります。

主な用途は次の2つです。

- 新規レシピ作成時の「マルチAI商品開発」
- 既存レシピ詳細画面の「AI改善提案」

どちらも単発の文章生成ではなく、複数の専門エージェントが調査・比較・検証を行い、最後に統括シェフエージェントが保存可能なレシピ案に統合します。

---

## できること

### 1. 新規レシピのAI商品開発

新規レシピ作成画面では、開発テーマを入力すると AI が商品案を生成します。

- 開発テーマから材料案と手順案を作成
- エージェント所見を表示
- 参照ソースを表示
- 生成結果をそのままフォームへ反映
- 生成後に続けて質問・再開発が可能

### 2. 既存レシピのAI改善提案

既存レシピ詳細画面では、表示中のレシピを読み取って改善案を作成します。

- 現在のレシピ内容をもとに改善案を生成
- エージェント所見を表示
- 参照ソースを表示
- 続けて質問・再改善が可能
- 改善案を「別レシピとして保存」または「このレシピに上書き保存」

### 3. 会話を踏まえた再評価

AI が出した案に対して追加質問した場合も、単なる雑な返答ではなく、再度エージェント群で調査・比較・検証を実施してから回答します。

- 新規作成側は商品開発文脈で再評価
- 改善側は既存レシピ改善文脈で再評価
- 必要なら proposal を更新
- 再提案依頼なら材料案・手順案を完全に作り直す

### 4. 採用結果を学習メモとして蓄積

AI が作成した案を実際に保存した時、その結果を次回以降の判断材料として使えるように蓄積します。

- 新規商品開発で保存した最終レシピを記録
- AI改善案を別レシピ保存した結果を記録
- AI改善案で既存レシピを上書きした結果を記録
- 生成時・改善時・再質問時の実行ログも別テーブルに記録
- 次回の生成・改善・再質問時に、近い採用事例を RAG 的にプロンプトへ注入

このため、使い続けるほど「過去に採用された配合・改善の傾向」を参照しながら提案精度を上げる構造になっています。

## エージェント構成

### 新規商品開発

基本の流れ:

1. Web調査エージェント
2. 海外・本場比較エージェント
3. レシピ統合エージェント
4. 食品科学エージェント
5. 科学検証エージェント
6. 統括シェフエージェント

会話継続時は、上記に加えて最終クロスチェックエージェントを通します。

### 既存レシピ改善

基本の流れ:

1. 料理文化調査エージェント
2. 調理技術調査エージェント
3. 海外・本場比較エージェント
4. 配合監査エージェント
5. 食品科学エージェント
6. 品質検証エージェント
7. 最終クロスチェックエージェント
8. 統括シェフエージェント

## プロバイダー

利用できる AI プロバイダー:

- `Groq`
- `Sakana AI（サブスク）`
- `Sakana AI（従量課金）`

デフォルトは `Groq` です。

### Sakana AI のロック

Sakana AI はコスト管理のため、クライアント側でロックされています。

- 未解除時は選択肢に `🔒` が表示される
- Sakana を選択した時に解除パスワードを要求
- 実行開始時にも再チェックする
- 未解除端末では保存済み設定が残っていても Groq 扱いに落とす

対象画面:

- 新規レシピ作成のマルチAI商品開発
- 既存レシピ詳細の AI改善提案
- それぞれの会話継続

注意:

- このロックはクライアント側の簡易ロックです
- 厳密な制御が必要な場合は Supabase Edge Function 側での制御が必要です

## 進行状況ポップアップ

AI エージェント実行中は、進行状況をポップアップ表示します。

表示対象:

- 新規商品開発の初回生成
- 新規商品案への追加質問・再開発
- AI改善提案の初回生成
- AI改善案への追加質問・再改善

表示内容:

- 現在の工程
- 進捗バー
- 全ステップ一覧

この表示はストリーミング実行結果そのものではなく、現在の処理モードに対応した工程表示です。

---

## 回答品質の方針

会話継続時も、次のルールで返答します。

- 一般論だけで済ませない
- 元レシピ、現在案、会話履歴に接続して答える
- 必要なら Web 調査を伴う
- 本場比較が必要な質問では海外ソースも見る
- 材料、分量、温度、時間、安全性の確認を重視する
- 更新しない場合でも、その理由を説明可能な形にする

---

## 保存挙動

### 新規作成側

- AI が作成した案はフォームに反映される
- 最終保存は通常の「レシピを保存」で行う
- 保存成功時に、その採用結果を学習メモへ記録する

### 改善側

- 別レシピとして保存
- 現在のレシピへ上書き保存
- どちらの保存でも、採用結果を学習メモへ記録する

閲覧専用レシピでは上書き保存は不可です。

## 現状の制約

- 進行状況表示は疑似ステップ表示であり、各 API 呼び出しの厳密なリアルタイム状態ではありません
- Sakana ロックはクライアント側制御です
- ESLint には既存の Hook 警告が残っています
- Groq / Sakana の API 利用量と応答速度はプロバイダー状況に依存します
- 学習は「保存された採用結果」を元にするため、自動で真偽判定する完全自己学習ではありません
