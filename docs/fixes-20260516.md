# 材料マスター・レシピ原価・数値入力の修正記録

**日付:** 2026-05-16  
**コミット:** `d1ecd6f`（`main`）  
**本番デプロイ:** GitHub Actions [Deploy to GitHub Pages #25955408753](https://github.com/MARUGO-s/app/actions/runs/25955408753)

## 概要

材料マスターが 1000 件で打ち切られて一覧・検索に出ない問題、レシピ入力で材料候補選択後に原価が入らない問題、全角数字入力への未対応、およびそれに伴う画面真っ白（ビルドエラー）を修正した。

---

## #1 材料マスターが 1000 件で止まる

### 症状

- 登録数がちょうど 1000 件前後のとき、それ以降に保存した材料が DB にはあるのに一覧・オートコンプリートに出ない
- 例: 「キャノーラ油」を登録しても検索ヒットしない

### 原因

`unitConversionService.getAllConversions()` が PostgREST のデフォルト上限（1000 行）で 1 回の取得が打ち切られていた。

### 修正内容

`src/services/unitConversionService.js` の `getAllConversions()` を、1000 件ずつ `range()` でページングし、全件を結合して返すように変更。

```js
const PAGE_SIZE = 1000;
// ...
.range(from, from + PAGE_SIZE - 1);
// rows.length < PAGE_SIZE になるまでループ
```

### 影響範囲

- データ管理 → 材料マスター一覧
- レシピ入力・パン入力の材料名オートコンプリート
- 在庫・原価計算で `getAllConversions()` を参照している処理

---

## #2 レシピ入力で材料選択後に原価が入らない

### 症状

- 材料マスターに登録済みの材料を候補から選んでも、仕入れ値・原価が空のままになることがある
- 特にマスター件数が多い／新規保存直後の材料で発生しやすい

### 原因（複合）

1. **AutocompleteInput:** 候補選択時に `onChange` が先に走り、材料名だけ先に更新される。その時点では `conversionMap` に未登録のため、参照・単価・原価がクリアされていた。
2. **RecipeFormIngredients:** 仕入れ値だけ入っていて原価が空の行について、旧データ判定が厳しく再計算されないケースがあった。
3. **conversionMap:** 画面表示時に一度だけ読み込んでおり、マスター更新後にマップが古いままになることがあった。

### 修正内容

| ファイル | 内容 |
|----------|------|
| `src/components/AutocompleteInput.jsx` | `onSelect` があるときは候補クリックで `onChange` を呼ばない（`onSelect` 側で名前・単価・原価を一括反映） |
| `src/components/RecipeFormIngredients.jsx` | 原価が空なら仕入れ値から再計算。候補選択後に `refreshConversionMap()`。フォーカス時にもマスターを再読込 |
| `src/components/RecipeFormBread.jsx` | 同上の数値正規化・候補選択フローに合わせて調整 |

## #3 全角数字の入力対応

### 症状

- 数量・仕入れ値・内容量などに全角数字（例: `１００`）を入れると、計算や保存が期待どおり動かない

### 修正内容

`src/utils/normalizeNumericInput.js` を追加・拡張。

- `toHalfWidthNumericString` — 全角数字・全角小数点・マイナス・カンマを半角に変換
- `normalizeNumericFieldValue` — フィールド名が数値系のときだけ半角化
- `parseNumericInput` — 計算用の数値パース
- `normalizeNumericInput` — `Input.jsx` 向けの別名 export

適用箇所:

- `src/components/Input.jsx`
- `src/components/RecipeFormIngredients.jsx`
- `src/components/RecipeFormBread.jsx`
- `src/components/IngredientMaster.jsx`
- `src/components/UnitConversionModal.jsx`

---

## #4 画面が真っ白になる（ビルドエラー）

### 症状

- 本番／ローカルでアプリ全体が白画面になり、コンソールに import エラー

### 原因

`Input.jsx` が `normalizeNumericInput` を import していたが、`normalizeNumericInput.js` 側で export されていなかった。

### 修正内容

`normalizeNumericInput.js` に `export const normalizeNumericInput = toHalfWidthNumericString` を追加。

## 確認手順（本番）

1. **材料マスター:** 1000 件超でも新規登録材料が一覧・検索に表示されること
